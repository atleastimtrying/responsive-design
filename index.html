What is responsive design?

design responsively, play with the browser and respond to what it shows you.

CSS Columns

- great response to tight measures - fallback isnt too terrible - allows for interesting flow of document. - dont let columns get too long

expect every element to scale

padding, line heights, measure of blocks of text. I think of a page as a collection of UI elements each responding in their own way. I’ve foudn this easier and more prgamatic, esp when you are starting ot have more unexpected pages.

use display inline-block

- font size 0 hack - expect boxes to scale - set to block on mobile - vertical-align: top;

use background images

- allows you to scale with css + media queries - background size:cover; - background-position:center; - great support.

use semantic grouping

wrap blocks of information into divs even if they arent used at some sizes, having them grouped In this way will be helpful. dont be afraid of rtl, allows for more interesting layouts (e.g left items being underneath right items later on).

CSS

  • display:block;
  • font-size:16x;
  • color:#333;
  • border-radius:2px;
  • padding:16px;

JS

  • document.ready();
  • document.queryselector();
  • arr.foreach();
  • el.className = 'foo';
  • el.style.display = 'none';

HTML

  • <title></title>
  • <header></header>
  • <article></article>
  • <section></section>
  • <footer></footer>

- they are tricky - require clear fixes - seem like a hack to me - floating eveything loses the natural flow of the document

Whitespace

when you get to mobile reduce white space, that means padding + margins + images. but NOT font size or line height

JS

window.innerWidth for js feature detection avoid using too much JS

Retina Displays

use vectors/icon fonts where possible. scale your images with CSS

Text Layouts

try not to worry about widows esp with CMS sites avoid br tags they break lines I don’t use ems for sizing my fonts, I find them confusing, I’d rather dictate which font sizes affect which bits. I can see how they would be useful though.

CSS Before/After

use css before/ afters for pseudo elements where possible. allows elements to be ignored/removed easily within a media query

Media Queries

I use lots of media queries to catch lots of changes within elements of the page rather than an iphone one or an ipad one. decide your largest and smallest sizes, respond to everything inbetween.

Absolute/fixed positioning

use absolute/fixed positioning at your peril, these do not scale well to mobile, try to think why you are using these techniques, can it be done using the normal flow of the document?

Obvious Questions

why not mobile first? people still think in terms of thedesktop, its just easier. there are less variants in mobile layouts than desktop ones. I’ve just found it easier overall!

why not a framework? I find frmaeworks hard to fix, I find them hard to work out just whats going on. they are fine as a starting point or for projects that have no exceptions. I have found most projects I come accross have exceptions pretty early on.

but x/y/z use floats, at least they are better than tables.. bully for them, I find floats are a horrible hack, they cause extra thinking to happen within a page when it’s not needed. HTML has a natural vertical flow in place, using a technique like inline-block allows you describe otehr elements like an image, it fits in the flow of the document.
